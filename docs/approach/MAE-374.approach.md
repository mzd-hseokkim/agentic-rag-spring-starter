## L1 Single (5줄)

- **변경 영역**: `examples/agentic-rag-demo/src/main/frontend/src/panels/MetricsPanel.tsx` 신규 1파일. `src/api/actuator.ts`의 `getMetric(name)` 헬퍼 그대로 사용 (MAE-370 산출물, 추가/수정 없음).
- **핵심 결정**: `useEffect` + `setInterval(5000ms)`로 폴링, 언마운트 시 `clearInterval` cleanup. 메트릭 카드는 `agentic.rag.llm.duration`(value/count 표시), `agentic.rag.retrieval.hits`(value 표시) 2개 고정 + 폴링 실패 시 마지막 성공값 유지 + "stale" 배지. 상태는 컴포넌트 로컬 `useState<Record<name, {value, stale}>>`.
- **검증**: 백엔드 기동(`./gradlew :examples:agentic-rag-demo:bootRun`) → ChatPanel ask 1회 호출 후 `llm.duration` 카드 값이 5초 이내 갱신되는지 수동 확인. 네트워크 차단 시 stale 배지 노출 확인.
- **리스크**: (1) 메트릭 미존재(404) 시 카드 자체를 숨길지 stale로 둘지 — 폴링 시작 전 1회 성공 전까지는 "loading", 이후 실패는 stale 유지. (2) Actuator exposure에 `metrics` 포함은 `application.yml:54` 확인 완료. `env`는 미포함이나 본 태스크 범위 외 (FR-5/MAE-373 책임).
- **롤백**: 파일 단일 추가이므로 `MetricsPanel.tsx` 삭제 + (통합되었다면) `App.tsx`의 import/렌더 1줄 제거로 원복.
