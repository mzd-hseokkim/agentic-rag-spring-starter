# Approach: MAE-372 - IngestPanel — Dropzone + URL 폼 + 결과 토스트

- **Level**: L1 (Single)
- **Source**: `docs/requirements/rag-playground-ui.requirements.md` (Sub-task 3, Wave 2)

## Approach (L1 Single)

- **변경 영역**: `examples/agentic-rag-demo/src/main/frontend/src/panels/IngestPanel.tsx` 신규 + 보조 `components/Dropzone.tsx`, `components/Toast.tsx`(또는 `useToast` 훅), 패널 CSS Module. `api/ingestion.ts`(MAE-370 산출물)의 `ingestFile`/`ingestUrl` 헬퍼만 호출.
- **핵심 결정**: 패널별 로컬 상태(`useState`/`useReducer`) — 전역 스토어 미도입. Dropzone은 의존 0의 native HTML5 drag/drop(`onDragOver`/`onDrop` + hidden `<input type=file>` fallback). 결과 토스트는 setTimeout 기반 자체 구현(자동 dismiss 3-4s, 성공/실패 색상 구분). 업로드 중복 방지 위해 in-flight 시 버튼 disabled.
- **검증**: `tsc --noEmit` 통과 + 데모 백엔드 기동 후 (1) 파일 drag&drop 업로드 → `IngestionResult` 토스트 표시 (2) URL 폼 submit → 동일 토스트 (3) 실패 경로(빈 URL/네트워크 오류) → 에러 토스트 확인.
- **리스크**: 다중 파일 / 대용량 파일 UX 미정 — 단일 파일·요구사항상 즉시 처리로 제한. 진행률 표시는 범위 외(요구 없음).
- **롤백**: 신규 파일 추가만 (App.tsx 통합은 Sub-task 6 책임) → `panels/IngestPanel.tsx`와 보조 파일 삭제로 revert.
