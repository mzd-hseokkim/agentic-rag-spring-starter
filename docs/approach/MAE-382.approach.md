# Approach: MAE-382 - event: ToolInvocationEvent 정의 + RagEventPublisher 통합 + 이벤트 테스트

- **Level**: L1 (Single)
- **Source**: `N/A — discover 생략` (parent MAE-376 description의 Acceptance Criteria + sibling MAE-378/379 산출물 가정)

## Approach (L1 Single)

- **변경 영역**: `agentic-rag-core/src/main/java/kr/co/mz/agenticai/core/common/event/ToolInvocationEvent.java` 신규 (sealed-free `RagEvent` 하위 marker interface — `LlmEvent` 패턴 답습) + 두 record `Started`/`Completed`. sibling MAE-378의 `FileSystemToolCallbackProvider`에서 각 `@Tool` 메서드 진입/종료 시 `RagEventPublisher.publish()` 호출 (try-finally로 종료 보장, 예외 시 `error`/`truncated` 필드 채움). 테스트는 `agentic-rag-core/src/test/.../ToolInvocationEventTest.java` 신규 + `FileSystemToolCallbackProvider` 단위 테스트에서 `ApplicationEventPublisher` 캡처로 4종 tool × (start+end) 발행 검증.
- **핵심 결정**: `RagEvent` 자체가 sealed가 아니므로(`RagEvent.java` Javadoc 명시) "sealed 확장" 표현 대신 **기존 marker interface 패턴**(`LlmEvent`, `RetrievalEvent`)을 그대로 따른다 — `ToolInvocationEvent extends RagEvent` + 내부 `record Started/Completed`. 필드: `toolName`, `argsSummary`(민감정보 절단된 요약), `startedAt`(Started/Completed 공통 식별자 역할), `durationMillis`(Completed only), `truncated`(boolean), `error`(String, nullable — Throwable.getMessage). 비동기 발행은 `RagEventPublisher` SPI 계약이 이미 "non-blocking" 명시이므로 기본 `ApplicationEventRagEventPublisher`를 그대로 사용 — tool 메서드 측에서 별도 스레드 분기 금지 (publisher 책임).
- **검증**: `./gradlew :agentic-rag-core:test`로 (1) `ToolInvocationEvent.Started`/`Completed` record 동등성/필드 채움 (2) `ApplicationEventRagEventPublisher`로 published 이벤트 캡처 시 4종 tool 각각 정확히 2건(start/end) 수신 (3) Completed의 `error` 필드가 예외 발생 케이스에서 non-null (4) `truncated=true`가 출력 상한 초과 시 채워짐. sibling MAE-378의 provider가 아직 머지 전이면 본 작업은 `ToolInvocationEvent` 타입 + `ApplicationEventRagEventPublisher` 라운드트립까지로 제한하고, provider 통합 테스트는 MAE-378 머지 후 후속 commit으로 분리.
- **리스크**: sibling MAE-378(검토 중) 머지 타이밍에 따라 `FileSystemToolCallbackProvider` 통합 호출이 본 worktree에 없어 빈 통합 테스트가 됨 — `ToolInvocationEvent` 타입 + core 이벤트 라운드트립 테스트만 본 PR에 포함하고 provider 호출부 통합은 MAE-378 머지 후 별도 commit/PR. `argsSummary` 안에 PII/경로가 길게 들어갈 수 있음 — 250자 truncate(기존 출력 상한 컨벤션 답습)로 차단.
- **롤백**: 신규 파일 추가만(`ToolInvocationEvent.java`, 신규 test) + provider 측 publish 호출 라인 ≤4 라인 — 해당 파일 삭제 + provider commit revert로 1:1 revert 가능.
