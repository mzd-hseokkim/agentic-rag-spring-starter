# Approach: MAE-370 - API 클라이언트 모듈 (src/api/) — fetch + EventSource 헬퍼

- **Level**: L1 (Single)
- **Source**: N/A — discover 생략 (Subtask, 상위 MAE-369의 범위 분해)

## Approach (L1 Single)

- **변경 영역**: `examples/agentic-rag-demo/src/main/frontend/src/api/` 신규 — `client.ts`(공용 fetch 래퍼), `rag.ts`(ask/askStream), `ingestion.ts`(ingestFile/ingestUrl), `actuator.ts`(getEnv/getMetric), `types.ts`(백엔드 정합 타입), `index.ts`(배럴).
- **핵심 결정**: 상대 경로 기반 fetch + 표준 `EventSource`(named event `TokenChunk`/`Completed`/`CitationEmitted`/`Failed` 분기), React/상태관리 의존 없음. 타입은 `agentic-rag-core` 레코드(`RagResponse`/`Citation`/`RagUsage`/`IngestionResult`) 필드에 1:1 매핑하여 수기 선언 (TS 자동생성 미도입).
- **검증**: `tsc --noEmit` 통과 + 데모 백엔드(`RagController`) 기동 후 수동 호출 4종(ask, askStream, ingestFile, ingestUrl) 200/스트림 수신 확인.
- **리스크**: `EventSource`는 GET만 지원하고 헤더 주입 불가 — 인증이 필요해지면 폴리필/POST-SSE로 교체. 현재 데모에는 인증 없음.
- **롤백**: 신규 파일 추가만 (기존 파일 무수정) → 디렉터리 삭제로 revert 가능.
