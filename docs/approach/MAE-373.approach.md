# Approach: MAE-373 - SettingsPanel — Actuator env 4 토글 + 활성 프로파일

- **Level**: L1 (Single)
- **Source**: `docs/requirements/rag-playground-ui.requirements.md` (FR-5)

## Approach (L1 Single)

- **변경 영역**: `examples/agentic-rag-demo/src/main/frontend/src/panels/SettingsPanel.tsx` 신규 + `App.tsx` 마운트 1줄 추가. API는 기존 `src/api/`의 `getEnv` 헬퍼 재사용.
- **핵심 결정**: 마운트 시 `GET /actuator/env` 1회 호출 → `propertySources`를 평탄화해 `agentic-rag.agents.enabled` / `factcheck.enabled` / `retrieval.query.hyde.enabled` / `retrieval.query.multi-query.enabled` 4개 키와 `spring.profiles.active`를 추출, `disabled` 체크박스 4개 + 활성 프로파일 텍스트로 read-only 렌더. 로컬 `useState` + `useEffect`만 사용 (글로벌 스토어 없음).
- **검증**: `application.yml`의 4개 토글 값과 화면 표시가 일치(수동 시연). 프로파일 미설정 시 `default` 표기. 로딩/에러 상태 텍스트 표시.
- **리스크**: Actuator `env` 엔드포인트 노출 누락 시 4xx — `management.endpoints.web.exposure.include`에 `env` 포함 여부 확인 필요(이미 노출 가정, 미노출이면 application.yml 1줄 추가는 본 작업 범위 외 후속).
- **롤백**: 컴포넌트/마운트 라인 revert로 즉시 원복 가능.
