## Approach (L1 Single)

- **변경 영역**: `agentic-rag-core/src/test/java/.../common/tool/` 신규 테스트 2종 — `WorkspaceSandboxTest`(상대경로/절대경로/symlink/Windows junction/`.gitignore` 매칭) + `OutputLimitsTest`(maxResults / maxBytes readFile / maxChars 라인 절단 + truncation 라인). main 머지 선반영(현재 feature/MAE-381은 main보다 10 commits 뒤 — 작업 진입 시 main rebase 필요).
- **핵심 결정**: MAE-345/368 테스트 답습 — 외부 컨테이너 없이 JUnit5 + AssertJ + `@TempDir`, Windows-only 케이스는 `@EnabledOnOs(OS.WINDOWS)` + `Runtime.exec("cmd /c mklink /J")` junction 생성; `.gitignore`는 MAE-377 자체 매처 동작 기준으로 positive/negative 페어 검증; output 테스트는 `OutputLimits` record 경유 — 사전 truncation 메서드가 없으면 헬퍼는 본 테스트 클래스 내부에 한정(테스트 전용).
- **검증**: `./gradlew :agentic-rag-core:test --tests *WorkspaceSandboxTest --tests *OutputLimitsTest` 그린; 모든 negative case에서 `SandboxException` (또는 명시적 거부 결과) `assertThatThrownBy`; positive case에서 `Path` 정상 resolve.
- **리스크**: Windows junction은 CI runner 권한/심볼릭 정책에 의해 실패 가능 → `@EnabledOnOs` + `assumeTrue(canCreateJunction)`으로 skip 처리; `.gitignore` 미지원 패턴은 MAE-377 Open Question(P2) 범위 — 본 테스트는 지원 패턴만 검증.
- **롤백**: 테스트 파일 2종 신규 추가만 — production 코드 무변경, 단일 revert 안전.
