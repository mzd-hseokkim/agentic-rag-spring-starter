# Approach: MAE-371 - ChatPanel — 메시지 + 입력 + sessionId + Stream/Sync 토글 + Citations

- **Level**: L1 (Single)
- **Source**: `docs/requirements/rag-playground-ui.requirements.md` (Sub-task 2, Wave 2)

## Approach (L1 Single)

- **변경 영역**: `examples/agentic-rag-demo/src/main/frontend/src/panels/ChatPanel.tsx` 신규 + 보조 `components/MessageList.tsx`, `components/CitationList.tsx`, 패널 CSS Module. `api/chat.ts`(MAE-370 산출물)의 `ask`/`askStream` 헬퍼만 호출.
- **핵심 결정**: 메시지 상태는 `useReducer`(actions: `userSubmit`/`assistantStart`/`tokenAppend`/`assistantDone`/`error`)로 토큰 append race-free. EventSource는 `useRef`에 보관하여 새 요청·언마운트 시 `close()` cleanup. Stream/Sync 토글은 패널 로컬 `useState` boolean. 응답 완료 후 `citations`가 빈 배열이면 "no sources" 텍스트, 아니면 `source` 파일명 + 단축 documentId(앞 8자) 인라인 chip 표시.
- **검증**: `tsc --noEmit` 통과 + 데모 백엔드 기동 후 (1) Stream 모드에서 "내 이름은 홍길동입니다" → "방금 알려준 내 이름은?" 멀티턴(동일 sessionId) 통과 (2) SSE 토큰이 한 글자씩 누적 표시 (3) Sync 모드에서 동일 질문이 한 번에 표시 (4) Citations chip 또는 "no sources" 표시.
- **리스크**: SSE 중간 끊김 시 부분 응답 잔존 — `error` 액션으로 상태 정리하되 재시도는 사용자 재입력에 위임(요구사항 외). 토큰 누적 중 React 18 batching은 reducer 패턴으로 회피.
- **롤백**: 신규 파일 추가만 (App.tsx 통합은 Sub-task 6 책임) → `panels/ChatPanel.tsx`와 보조 파일 삭제로 revert.
