# Approach: MAE-380 — autoconfigure: AgenticRagFsToolsAutoConfiguration + 통합 테스트

- **Parent**: MAE-376 (FS Tools 통합)
- **Depends on**: MAE-377 (`WorkspaceSandbox` SPI), MAE-378 (`FileSystemTools` / `FileSystemToolCallbackProvider`), MAE-379 (`AgenticRagProperties.tools.fs`)
- **Source**: N/A — discover 생략 (Subtask, 부모 MAE-376의 분해 산출물)

## Approach (L1 Single)

- **변경 영역**: `agentic-rag-autoconfigure/src/main/java/.../AgenticRagFsToolsAutoConfiguration.java` 신규 1개 + `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`에 1줄 추가 + 통합 테스트 `src/test/java/.../AgenticRagFsToolsAutoConfigurationTest.java` 신규 1개. 기존 `AgenticRagToolsAutoConfiguration`은 손대지 않음 — `CatalogToolProvider`가 `ToolCallbackProvider` Bean을 자동 집계함.
- **핵심 결정**: `@AutoConfiguration(before = AgenticRagCoreAutoConfiguration.class)` + `@ConditionalOnClass(ToolCallbackProvider.class)` + `@ConditionalOnProperty(prefix="agentic-rag.tools.fs", name="enabled", havingValue="true")` (matchIfMissing=false — silent default 금지). `WorkspaceSandbox` Bean은 `props.getTools().getFs().getRoot()`가 비어 있으면 `IllegalStateException`로 빠른 실패; root가 있으면 `Path.of(root)` 기반 기본 구현을 등록. `FileSystemToolCallbackProvider` Bean은 `FileSystemTools(sandbox, outputLimits)` 생성 후 래핑. 두 Bean 모두 `@ConditionalOnMissingBean`로 사용자 override 허용.
- **검증**: `ApplicationContextRunner` 기반 단위 통합 테스트 3 케이스 — (1) `agentic-rag.tools.fs.enabled=true` + `root=<tmp>` → `WorkspaceSandbox` + `FileSystemToolCallbackProvider` Bean 존재, `CatalogToolProvider`가 4종 tool 노출 (`fs_read`, `fs_write`, `fs_list`, `fs_stat`). (2) `enabled=false` (또는 미설정) → 두 Bean 모두 미생성. (3) `enabled=true` + `root` 미지정 → 컨텍스트 기동 실패(`IllegalStateException`).
- **리스크**: (1) `FileSystemTools`의 정확한 생성자 시그니처는 MAE-378 산출물에 의존 — 의존 task 미머지 시 본 task 구현 블록. start 시점에서 MAE-378 브랜치/PR 머지 상태 확인 필요. (2) MAE-377의 `WorkspaceSandbox` 위치가 `agentic-rag-core`로 이동 예정 — 본 task 시점에서는 MAE-378의 임시 위치(`tools.fs` 패키지)와 core 둘 중 어느 import를 쓸지 확정 필요. core 머지 완료면 core 사용, 아니면 임시 위치 사용 후 후속 task에서 import 경로만 갱신.
- **롤백**: 파일 2개 신규 + imports 1줄 추가뿐이므로 `AgenticRagFsToolsAutoConfiguration.java` 삭제 + imports 라인 제거 + 테스트 파일 삭제로 완전 원복. 기존 `AgenticRagToolsAutoConfiguration` 무수정이므로 회귀 없음.
