# Approach: MAE-383 - demo 앱에 fs tools 통합 — application-tools.yml + README 시연 시나리오 + IT E2E

- **Level**: L1 (Single)
- **Source**: `N/A — discover 생략, Jira description 기반`

## Approach (L1 Single)

- **변경 영역**: `examples/agentic-rag-demo` 3파일 — `application-tools.yml`(fs 블록 추가), `README.md`(시연 시나리오 1줄), 신규 `IT_FileSystemTools` E2E 테스트.
- **핵심 결정**: 기존 패턴 답습 — `application-tools.yml`에 `agentic-rag.tools.fs` 블록을 properties 키(`enabled`/`root`/`respect-gitignore`/`max-read-bytes`/`max-read-lines`/`max-list-entries`)에 맞춰 추가하고, IT는 기존 `IT_RollingSummaryPolicy` 스타일(`@SpringBootTest` + `@ActiveProfiles("tools")`)을 그대로 따른다. `root: ../../docs` 상대경로, `DemoConfig` 보정 로직 없음.
- **검증**: `IT_FileSystemTools` E2E 통과 — `CatalogToolProvider`가 `FileSystemToolCallbackProvider`를 집계함 + `fs_grep` 툴콜 정상 동작 + `../` sandbox 경로 탈출 차단 3건 검증.
- **리스크**: `root: ../../docs` 상대경로가 테스트 작업 디렉토리 기준으로 해석되어 IT에서 docs/ 미해석 가능 — IT는 프로퍼티 오버라이드 또는 절대경로 fixture로 회피.
- **롤백**: revert 가능 — 데모 한정 신규 추가분이며 라이브러리 모듈/기존 프로파일 무영향.
