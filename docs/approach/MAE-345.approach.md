## Approach Summary (L2 Story)

Query path(`retrieval → agent → LLM → factcheck`)에 Spring Boot 3 Micrometer Observation API를 사용해 일관된 OTel span 트리를 구성한다. RAG 도메인 attribute(retriever id, k, score, fusion strategy, verdict, confidence)를 표준화하고, streaming 토큰은 span event로 발행한다. correlation-id는 baggage로 전파한다. Ingestion 측 instrumentation 및 exporter 구현은 비범위(autoconfig 사용).

## Architecture

- **신규**: `core` 내 `RagObservability` (attribute key 상수 + ObservationConvention enum), `RagBaggage`(correlation-id 헬퍼).
- **수정**: `retrieval`(VectorRetriever / BM25Retriever / FusionRetriever / Reranker 각 단계 Observation 래핑), `agents`(Planner / Synthesizer / ToolCall flow span), LLM 호출 wrapper(`core` 또는 `agents`), `factcheck`(verdict span).
- **의존 방향**: 모든 모듈 → `core`만 참조. `io.micrometer:micrometer-tracing-bridge-otel`은 `core`의 `compileOnly` + 사용자 starter optional dependency. SPI 원칙 유지.
- **이벤트 연결**: 기존 `RagEventPublisher` 흐름과 독립 — Observation은 cross-cutting trace, RagEvent는 도메인 이벤트. 중복 발행하지 않는다.

## Key Decisions

| # | 결정 | 대안 | 선택 이유 |
|---|---|---|---|
| 1 | Micrometer `Observation` API 사용 | OTel SDK 직접 호출 | Spring Boot 3 표준. autoconfig로 OTel/Zipkin/Jaeger 모두 호환. Spring AI 1.x 관찰성 빌트인과 일관 |
| 2 | attribute prefix `rag.*` | OTel semconv `db.*` 재사용 | RAG 도메인 전용 의미. semconv 미정 영역 — 충돌 회피 |
| 3 | streaming 토큰을 span event로 | child span | per-token span은 cost 폭증. event는 비차단 + 충분한 가시성 |
| 4 | correlation-id를 baggage로 | MDC만 사용 | 비동기/스트리밍 경계에서 MDC 끊김. baggage가 OTel context와 함께 전파됨 |
| 5 | tracing 라이브러리 `compileOnly` | implementation | starter 사용자가 OTel 미사용 시 의존성 강제 회피. ConditionalOnClass로 자동 비활성 |
