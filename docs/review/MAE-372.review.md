### Info

- **[Info] `useToast.ts:11` — module-level `_id` counter**
  - 모듈 전역 카운터를 사용. 단일 페이지 SPA에서는 무해하나, 동일 모듈이 여러 React root에 마운트되면 ID 공유. 현재 Playground 단일 root 가정에 부합하므로 조치 불필요. 향후 SSR/멀티 root 도입 시 `useRef` 또는 `crypto.randomUUID()` 검토.
- **[Info] `Dropzone.tsx:46` — 키보드 활성화 시 default 방지 누락**
  - Space 키 입력 시 `e.preventDefault()` 없이 `inputRef.current?.click()` 호출. 페이지 스크롤이 함께 발생할 수 있음. role="button" 일관성을 위해 `e.preventDefault()` 추가 권장 (비차단).

### Strengths (Positive Notes)

- ARIA 속성 적절: Dropzone `role="button"` + `aria-label`, ToastList `role="status" aria-live="polite"`, 닫기 버튼 `aria-label="닫기"`.
- `busy` 가드를 핸들러 진입부와 UI(`disabled`) 양쪽에 두어 race condition 방지.
- 에러 메시지가 `err instanceof Error` 분기로 안전하게 직렬화됨.
